## How to Ingest Manuals into CrankDoc

Once you have a PDF downloaded, use the ingestion script to add it to the RAG vector database:

```bash
# Example: Ingest a Honda CBR600RR service manual
npx tsx scripts/ingest-document.ts \
  --file ./data/manuals/honda-cbr600rr-service-2003.pdf \
  --make Honda \
  --model CBR600RR \
  --year-start 2003 \
  --year-end 2004 \
  --type service_manual

# Example: Ingest a Kymco AK 550 service manual
npx tsx scripts/ingest-document.ts \
  --file ./data/manuals/kymco-ak550-service.pdf \
  --make Kymco \
  --model "AK 550i" \
  --year-start 2017 \
  --type service_manual

# Example: Ingest a BMW R1250GS rider's manual
npx tsx scripts/ingest-document.ts \
  --file ./data/manuals/bmw-r1250gs-riders-manual.pdf \
  --make BMW \
  --model R1250GS \
  --year-start 2019 \
  --type owners_manual
```

### What the script does:
1. Parses the PDF and extracts text (motorcycle-manual-aware parsing)
2. Chunks the text into searchable sections
3. Generates vector embeddings via OpenAI
4. Stores everything in Supabase (`document_sources` + `document_chunks`)
5. Links to the matching motorcycle in the database

### Supported manual types:
- `service_manual` — workshop/repair manuals
- `owners_manual` — owner's/rider's manuals
- `parts_catalog` — parts lists with diagrams
- `tsb` — technical service bulletins

### After ingesting, verify with:
```bash
npx tsx scripts/verify-vector-data.ts
```

### Suggested folder structure:
```
data/
  manuals/
    honda-cbr600rr-service-2003.pdf
    honda-cbr600rr-owners-2024.pdf
    yamaha-mt07-service-2014.pdf
    bmw-r1250gs-riders-2021.pdf
    kymco-ak550-service.pdf
    ...
```

Note: The `data/manuals/` directory is gitignored — manuals are not committed to the repo.
